## 1. 정보 구조 (IA) 및 네비게이션

- 3개 탭: **나의 냉장고 / 레시피 추천 / 마이페이지**
- **모바일(320px~767px)**: 하단 탭 네비게이션 고정 (높이 56px)
- **웹/데스크톱(1024px 이상)**: 상단 네비게이션 바로 전환 (높이 64px, 활성 탭은 Steel Blue 밑줄로 표시)
- 사이드바 방식은 채택하지 않음 (UI-UX-Guideline.md 4장 참고)

## 2. 화면별 스펙

### 2.1 나의 냉장고 (`/fridge`)
**목록 화면**
- 카테고리별 섹션(채소/육류/유제품/조미료/새로 추가됨 등), 섹션 헤더 + 아이템 리스트
- 각 아이템: 토글(체크박스 아이콘, 24×24px) + 재료명, 클릭 시 즉시 보유 여부 반전
- 하단(또는 리스트 끝) "재료 추가" 버튼(Primary, Steel Blue)

**재료 추가 화면 (검색 기반, 브라우징 아님)**
- 진입 시 검색창만 노출, 결과 리스트는 비어있는 상태("검색어를 입력하면 재료를 찾을 수 있어요")
- 타이핑 즉시 부분 문자열 매칭 결과 노출 (디바운스 150~200ms 권장)
- 검색 결과 유무와 무관하게 리스트 최하단에 **`"{검색어}" 직접 추가하기`** 항목 항상 노출
- 결과 없을 때: "검색 결과가 없어요. 아래에서 직접 추가할 수 있어요" 안내 문구 + 직접 추가 항목
- 항목 클릭으로 다중 선택(선택 시 배경 `#DCE6EC` + 체크 아이콘), 하단 "선택한 재료 추가 (N)" 버튼으로 일괄 confirm
- confirm 시 `POST /api/fridge` 호출 (마스터 재료는 `ingredient_id`, 커스텀은 `custom_name`으로 구분 전송)

**유통기한 (P1)**
- 재료 등록 시 카테고리 기준 기본 유통기한을 자동 계산해 표시, 사용자가 직접 수정 가능
- 목록에서 유통기한 임박 재료는 시각적으로 강조(별도 색상 추가 없이, 텍스트 굵기/작은 라벨로 구분 — 가이드라인의 "색상 최소화" 원칙 유지)

### 2.2 레시피 추천 (`/recipes`)
**목록 화면**
- 상단 필터: 전체 / 완전 보유 / 일부 부족 (Steel Blue 채움 vs 윤곽선으로 활성 상태 구분)
- 카드 그리드: 모바일 1~2열, 태블릿 2열, 데스크톱 3열
- 카드 구성: 사진(16:9) + 요리명 + 매칭 배지(완전 매칭: 채움 배지 / 일부 부족: 윤곽선 배지)
- 정렬 기본값: 완전 매칭 우선

**상세 화면**
- 뒤로가기 버튼, 요리명, 즐겨찾기 아이콘(하트, 클릭 시 `POST/DELETE /api/recipes/:id/favorite`)
- 매칭 배지, 보유 재료 태그 목록, 부족 재료 태그 목록(부족 재료 없으면 섹션 자체 숨김)
- 조리 순서(번호 목록)
- 영양 정보 표시 (P1, F14): 1인분 기준 칼로리/탄단지/나트륨 — `recipes` 테이블의 nutrition 필드 그대로 노출
- 하단 "이 요리 완성했어요" 버튼 → `POST /api/recipes/:id/complete` 호출, 성공 시 버튼 텍스트 일시적으로 "완성 기록 저장됨"으로 피드백 후 원복
- 진입 시 `POST /api/recipes/:id/view` 자동 호출 (최근 조회 기록)

### 2.3 마이페이지 (`/mypage`)
- **최근 조회한 요리**: 가로 스크롤 카드(최대 5개), 클릭 시 레시피 상세로 이동
- **즐겨찾기한 요리**: 세로 리스트, 하트 아이콘으로 즐겨찾기 해제 가능
- **이번 달 인사이트**: "가장 자주 만든 요리 TOP 3", "가장 자주 소진된 재료 TOP 3" — 텍스트+횟수 위주(그래프 사용 안 함)
- 데이터 없을 때 각 섹션에 빈 상태 문구 표시 ("아직 조회한 요리가 없어요" 등)
- **하루 섭취량 트래킹 (P1, F15)**: 오늘 완성한 요리들의 칼로리 합산 표시
- **목표 기반 필터 (P2, F16)**: `user_profiles.goal` 설정 시 레시피 추천 화면에 필터 옵션 추가. 의학적 식이 제한은 다루지 않는다는 안내 문구 필수 노출

## 3. 상태 관리

- 서버 상태(냉장고 목록, 레시피 목록, 마이페이지 요약 등): React Query 또는 SWR 같은 서버 상태 라이브러리 사용 권장 (캐싱, 재검증 처리 간편)
- 로컬 UI 상태(검색어 입력값, 선택된 필터, 모달 열림 여부 등): 컴포넌트 로컬 `useState`
- 전역으로 공유해야 하는 상태(로그인 사용자 정보): Supabase Auth 세션을 Context로 감싸 최상위에서 제공

> 가정: 별도 전역 상태 라이브러리(Redux, Zustand 등)는 이 서비스 규모에서 불필요하다고 판단하여 도입하지 않는 것으로 가정함. 추후 상태가 복잡해지면 재검토.

## 4. 디자인 시스템 요약 (상세는 UI-UX-Guideline.md 참고)

- **색상**: Steel Blue(`#4A7C9E`) 단일 액센트 + 아이스 그레이 뉴트럴 톤. 상태 구분은 색상이 아니라 채움/윤곽선/톤 농도로 표현
- **타이포그래피**: Inter, 8px 기준 그리드
- **터치 대상**: 최소 44×44px
- **명도 대비**: 본문/버튼 4.5:1 이상 목표 (일부 톤은 경계값이므로 실제 구현 시 WebAIM 등으로 재검증 권장)

## 5. API 연동 매핑 (BACKEND.md 5장 참고)

| 화면/기능 | 호출 API |
|---|---|
| 나의 냉장고 목록 | GET /api/fridge |
| 재료 검색 | GET /api/ingredients/search |
| 재료 추가 | POST /api/fridge |
| 재료 보유 토글/유통기한 수정 | PATCH /api/fridge/:id |
| 레시피 목록 (필터 포함) | GET /api/recipes |
| 레시피 상세 | GET /api/recipes/:id |
| 레시피 조회 기록 | POST /api/recipes/:id/view |
| 즐겨찾기 추가/해제 | POST, DELETE /api/recipes/:id/favorite |
| 레시피 완성 처리 | POST /api/recipes/:id/complete |
| 마이페이지 요약 | GET /api/mypage/summary |

## 6. 컴포넌트 목록 (초안)

- `FridgeCategorySection` — 카테고리별 재료 목록 섹션
- `IngredientToggleItem` — 개별 재료 토글 아이템
- `IngredientSearchPanel` — 검색 기반 재료 추가 패널 (커스텀 추가 옵션 포함)
- `RecipeFilterBar` — 전체/완전 보유/일부 부족 필터
- `RecipeCard` — 레시피 카드 (매칭 배지 포함)
- `RecipeDetailView` — 레시피 상세 (즐겨찾기, 완성 버튼 포함)
- `IngredientTag` — 보유/부족 재료 태그 (공용 컴포넌트)
- `MyPageRecentList` / `MyPageFavoriteList` / `MyPageInsightCard` — 마이페이지 섹션별 컴포넌트
- `TopNav` (데스크톱) / `BottomTabNav` (모바일) — 반응형 분기 네비게이션
